## Команда агентов

Перед любой задачей — определи агента и думай как он.
Детали в `.claude/agents/*.md`.

| Агент | Когда вызывать |
|-------|---------------|
| **architect** | Новые модели, маршруты, изменения схемы БД |
| **design-keeper** | Новые UI-секции, проверка токенов, визуальный спек |
| **backend** | Controllers, Services, FormRequests, Inertia responses |
| **frontend** | React-компоненты, страницы, формы, анимации |
| **admin** | Orchid Screens, CRUD, управление контентом |
| **content** | Тексты (брать из docs/CONTENT.md) |
| **conversion-engineer** | Формы, CTA, аналитика, source-теги |
| **devops** | Сервер, деплой, SSL, домен |

**Быстрые команды:**
```
«сделай блок X»     → design-keeper (спек) → frontend (код)
«добавь форму»      → architect + backend + conversion-engineer + frontend
«добавь модель»     → architect → admin → backend
«задеплой»          → devops
«почему 500»        → читай storage/logs/laravel.log
```

## Контекст проекта

**Клиент:** ГБПОУ «Колледж декоративно-прикладного искусства им. Карла Фаберже», Москва

**Суть:** Промо-сайт — новое цифровое лицо колледжа. Современный, вдохновляющий, конкурирует с лучшими образовательными институциями страны. Не официальный сайт — промо.

**Главная задача сайта:** Привлечь абитуриентов → привести на День открытых дверей

**Главный CTA:** «Поступить» / «Записаться на ДОД»

**Слоган:** «Учим тех, кто создаёт шедевры»

**Тон:** Вдохновляющий, кураторский, как культурная институция. Не казённый, не академический. Говорим про будущее, не про учёбу.

**Очерёдность разработки:**
```
Очередь 1 → Абитуриенты    (сейчас — главная + специальности + поступление)
Очередь 2 → ДПО            (курсы, переподготовка)
Очередь 3 → Магазин        (e-commerce)
Очередь 4 → Партнёры + всё остальное
```

## Жёсткие правила проекта

1. **Все тексты — из `docs/CONTENT.md` дословно** — не переписывать, не сокращать
2. **H1 каждой страницы — из `docs/SEO.md`** — не менять
3. **Цвета и шрифты — только из `docs/DESIGN_SYSTEM.md`** через токены
4. **Border-radius: 0 везде** — прямые углы, без исключений
5. **Все тексты UI через Orchid / SiteSettings** — ничего не хардкодить
6. **Schema.org JSON** — вставить из `docs/SEO.md` в head каждой страницы
7. **ДОД-баннер** — управляется из Orchid, включается/выключается без разработчика
8. **Метрика** — из `SiteSettings::get('metrika_id')`, не хардкодить ID
9. **Source-теги на всех CTA** — см. CTA-карту в `docs/IA.md`
10. **Mobile-first** — все компоненты от 375px

**Запрещено:**
- «Уважаемые абитуриенты», «Добро пожаловать» — в любом тексте
- Цветные кнопки (синий, зелёный, красный)
- Градиенты в UI
- Скругления на карточках, кнопках, полях форм
- Хардкодить контакты, адреса, телефоны — всё через SiteSettings

## Маршруты (Очередь 1)

```php
// Публичные — после консолидации (см. docs/IA.md, решение от 2026-04-22)
GET  /                          → HomeController@index
GET  /abiturientam              → AbiturientController@index     // единственный хаб
GET  /specialnosti              → redirect('/abiturientam', 301)
GET  /specialnosti/{slug}       → SpecialtyController@show       // 9 страниц, канонический URL
GET  /news                      → NewsController@index
GET  /news/{slug}               → NewsController@show
GET  /contacts                  → ContactsController@index
GET  /privacy                   → PageController@privacy

// API — формы
POST /api/leads                 → LeadController@store          // главная форма
POST /api/leads/dod             → LeadController@storeDod       // запись на ДОД
POST /api/leads/courses         → LeadController@storeCourses   // подготовительные курсы

// Служебные
GET  /404                       → errors.404
/admin/*                        → Orchid
```
